# Диагностика репозитория collage

Ветка: `fix/topbar-single-owner`
База: `main` после отката на `2bcd8773799620d5eb4ad30ecd31c19fd0f7c3f1`.

## Что исправлено в этой ветке

### 1. Убран конфликт двух CSS-владельцев верхней панели

Было:

- `src/styles.css` управлял верхним layout: `.app-shell`, `.topbar`, `.app-topbar`, `.file-panel`, `.document-panel`, `.album-bar`, `.mode-strip`.
- `src/topbar.css` управлял теми же зонами вторым слоем.

Это создавало CSS-драку: любые новые правила с `!important` могли менять весь корневой layout и двигать рабочую область.

Решение:

- оставлен один владелец интерфейса: `src/styles.css`;
- из `src/main.jsx` удалён импорт `./topbar.css`;
- `src/topbar.css` удалён из ветки как дублирующий override-слой.

### 2. Cloud save больше не обязан кликать кнопку редактора

Было:

- `public/cloud-auth.js` искал кнопку `Сохранить` в DOM;
- кликал её;
- ждал таймер;
- потом читал localStorage.

Решение:

- добавлена функция `getEditorProject()`;
- сначала используется будущий React-мост `window.__collageApp.getProject()`, если он есть;
- если моста пока нет, используется последний локально сохранённый проект;
- обязательный DOM-клик и таймер убраны.

Важно: полноценный React bridge ещё нужно добавить в `AppLive.jsx` отдельным безопасным коммитом, когда файл можно править целиком без риска снести редактор.

### 3. Добавлена защита размера облачного проекта

Перед отправкой проекта в облако `public/cloud-auth.js` теперь собирает JSON payload и считает его размер.

Если payload больше `50 MB`, отправка не выполняется, а пользователь получает понятную ошибку: проект слишком большой, нужно очистить лишние фото или скачать JSON.

Это защищает от случайной отправки огромного base64-проекта в API.

Он запускает:

Первая проверка ветки после добавления workflow прошла успешно. После последней маленькой правки `cloud-auth.js` GitHub пока не отдал новый status-check через API, поэтому её нужно перепроверить в UI GitHub Actions или следующим run.

## Оставшиеся технические риски

### 1. React bridge для cloud save ещё не подключён со стороны AppLive

`public/cloud-auth.js` уже умеет использовать:

```js
window.__collageApp.getProject()
```

Но `AppLive.jsx` пока не экспортирует этот мост. До подключения моста облако сохраняет последний локально сохранённый проект.

### 2. Фото грузятся как dataURL без ограничения размера

В `uploadPhotos` нет лимита размера/количества. Большие альбомы могут раздувать localStorage, облачный JSON и экспорт. В этой ветке добавлена защита именно перед облачной отправкой, но не перед самой загрузкой фото в редактор.

### 3. Нет `package-lock.json`

Зависимости с `^` могут подтянуть разные версии при будущих установках. Лучше добавить lock-файл после локального `npm install`, но я не генерирую его вручную без запуска npm.

### 4. Layout рамок может переполниться

При маленьком холсте, большом зазоре и большом количестве фото-окон минимальный размер рамки может физически не поместиться.

## Рекомендованный порядок дальнейших правок

1. Проверить/смерджить эту ветку: она убирает CSS-конфликт, чинит cloud-save fallback, добавляет cloud-size guard и build-check.

3. Потом добавить лимиты фото при загрузке в редактор.
4. Потом аккуратно менять расположение верхних панелей уже через JSX-структуру, а не CSS-заплатками в конец файла.
