# README ДЛЯ ДАЛЬНЕЙШЕЙ РАБОТЫ НАД COLLAGE CREATOR

Этот файл — не публичная инструкция для пользователя.  
Это рабочая памятка для нас, чтобы дальше не превращать редактор в CSS-пирог и не ломать уже собранную механику.

---

## 1. Что это за проект

`Collage Creator` — редактор фотоальбома.

Главная задача проекта:

> Быстро собирать фотоальбом по страницам, добавлять текст, смотреть развороты и экспортировать домашнюю брошюру под печать А4 → А5.

Это не просто коллажер.  
Это редактор с несколькими слоями логики:

1. **Страницы альбома** — обычный порядок чтения.
2. **Разворот** — две соседние страницы рядом.
3. **Брошюра** — физический порядок печати листов.
4. **Текстовые слои** — отдельный слой поверх страницы.
5. **Экспорт** — PNG страницы, PNG разворота, PNG/ZIP брошюры.

---

## 2. Главный принцип архитектуры

Источник правды — это обычные страницы альбома:

```text
1 2 3 4 5 6 7 8
```

Брошюра НЕ должна переставлять реальные страницы внутри проекта.

Брошюра — это только другое представление этих же страниц:

```text
Лист 1 лицевая: [8][1]
Лист 1 оборот:  [2][7]
Лист 2 лицевая: [6][3]
Лист 2 оборот:  [4][5]
```

То есть:

```text
редактор страниц = создаёт контент
режим брошюры = показывает, как этот контент печатать
```

## 3. Основные файлы

### `src/AppLive.jsx`

Главный React-файл.

В нём находится:

- состояние проекта;
- страницы альбома;
- режимы `single / spread / booklet`;
- работа с фото-рамками;
- экспорт PNG;
- экспорт брошюры;
- логика пустых страниц;
- drag-and-drop страниц;
- подключение текстовых слоёв к брошюре.

Этот файл большой.  
Перед изменениями обязательно искать существующую функцию, а не добавлять дубликат.

---

### `src/editor/booklet.js`

Чистая логика брошюры.

Там НЕ должно быть:

- React;
- DOM;
- Konva;
- CSS;
- localStorage;
- скачивания файлов.

Только расчёт:

```js
buildBookletPlan({ pageCount, sheetsPerBlock })
```

Пример для 8 страниц / 2 листа в блоке:

```text
[8][1]
[2][7]
[6][3]
[4][5]
```

Если нужна новая логика порядка листов — сначала править здесь, а уже потом UI.

---

### `src/styles.css`

Главный CSS.

Сейчас он должен быть цельным, без старых patch-пирогов.

Запрещено добавлять в конец файла бесконечные блоки вида:

```css
/* Patch 22 */
...
/* Patch 23 */
...
/* Fix fix final */
...
```

Если правка относится к существующей области — редактировать существующий раздел.

Если раздела не хватает — создать один нормальный раздел с понятным названием, а не временную заплатку.

---

### `public/album-layers.js`

DOM-слой для режимов:

- `Текст`;
- `Рисунки`;
- `Шаблоны`.

Важно: это старый внешний слой, который пока живёт рядом с React.

Он отвечает за:

- панель текстов;
- панель настроек текста;
- сохранение `extraLayers`;
- передачу текстов в проект.

Нельзя бездумно ломать этот файл, потому что текстовые слои используются ещё и в экспорте брошюры через `extraLayers`.

---

### `public/album-layers.css`

CSS для DOM-слоя текста/рисунков/шаблонов.

Важно:

В режиме `Текст` старая фотопанель должна скрываться.

Правильная схема:

```text
редактор текста | страницы | холст | настройки текста
```

Неправильная схема:

```text
фото | страницы | холст
```

---

### `vite.config.js`

Должен быть чистым:

```js
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [react()],
});
```

Нельзя возвращать туда string-replace-костыли.

Если `AppLive.jsx` не собирается — чинить `AppLive.jsx`, а не сборщик.

Есть два разных уровня режимов.

### Уровень 1: чем редактируем страницу

```text
Коллаж
Текст
Рисунки
Шаблоны
```

Это инструмент редактирования.

---

### Уровень 2: как смотрим страницы

```text
Страница
Разворот

```

Это способ просмотра/экспорта.

Нельзя смешивать эти уровни в одну кучу.

## 5. Страница, разворот, брошюра

### `Страница`

Одна страница альбома.

Используется для обычного редактирования.

---

### `Разворот`

Две соседние страницы:

```text
[1][2]
[3][4]
[5][6]
```

Используется для просмотра альбома как разворота.

---

### `Брошюра`

Физическая печать.

Для 8 страниц / 2 листа:

```text
Лист 1 лицевая: [8][1]
Лист 1 оборот:  [2][7]
Лист 2 лицевая: [6][3]
Лист 2 оборот:  [4][5]
```

В режиме брошюры пользователь должен понимать:

- какой блок;
- какой лист;
- лицевая или оборот;
- какие страницы рядом;
- сколько пустых страниц нужно до полного блока.

---

## 6. Пустые страницы

Пустая страница — это реальный тип страницы:

```js
page.isBlankPage === true
```

Это НЕ страница с пустыми фото-окнами.

Пустая страница:

- белая;
- без фото-рамок;
- нужна для добивки блока брошюры;
- в редакторе может показывать надпись “Пустая страница”;
- в PNG для печати должна быть просто белой.

---

## 7. Текстовые слои

Тексты живут отдельно от рамок и фото.

Они хранятся в `extraLayers`.

Важно:

- при добавлении страницы слои должны сдвигаться;
- при удалении страницы слои должны сдвигаться;
- при перетаскивании страниц текст должен ехать вместе со страницей;
- при экспорте брошюры текст должен попадать на нужную физическую сторону.

Нельзя хранить текст только визуально в DOM без попадания в проект/JSON.

---

## 8. Экспорт

Есть несколько типов экспорта.

### PNG страницы

Экспортирует одну страницу.

---

### PNG разворота

Склеивает две страницы в один файл без зазора.

---

### PNG брошюры

Экспортирует физическую сторону листа:

```text
[8][1]
```

или:

```text
[2][7]
```

---

### ZIP пакет печати

Должен содержать:

```text
README_PRINT_ORDER.txt
print-order.csv
booklet-manifest.json
print-preview.html

block-01/
  booklet-block-01-sheet-01-front.png
  booklet-block-01-sheet-01-back.png
```

ZIP нужен, чтобы не скачивать десятки PNG вручную.

## 9. Что запрещено

### 9.1. Запрещены CSS-пироги

Нельзя добавлять:

```css
/* patch 22 */
...
/* patch 23 hotfix */
...
/* final fix */
...
/* final final */
...
```

Правильно:

- найти существующий раздел;
- исправить правило там;
- удалить старое конфликтующее правило;
- не оставлять два правила, которые спорят друг с другом через `!important`.

---

### 9.2. Запрещены `nth-child` для подписей кнопок

Нельзя делать так:

```css
.some-button:nth-child(3)::after {
  content: "Сетка";
}
```

Почему:

- добавилась новая кнопка;
- порядок детей изменился;
- подписи уехали;
- “+ Страница” стала “Копия”;
- “Направляющие” стала “Сетка”.

Подпись кнопки должна быть в JSX, а не в CSS.

---

### 9.3. Запрещено скрывать настоящий текст кнопки и подставлять `::after`

Нельзя:

```css
button {
  font-size: 0;
}

button::after {
  content: "Копия";
}
```

Это ломает интерфейс и доступность.

---

### 9.4. Запрещены временные файлы-патчи

Нельзя добавлять отдельные файлы типа:

```text
src/patch20-top-panels.css
src/fix-final.css
src/temp-ui.css
```

Если нужен новый CSS — он должен быть частью нормальной структуры `src/styles.css`.

---

### 9.5. Запрещено чинить исходник через Vite

Нельзя возвращать в `vite.config.js` плагины, которые делают string replace в `AppLive.jsx`.

Если JS сломан — чинить JS.

---

### 9.6. Нельзя смешивать разные уровни управления

Нельзя в одну строку без структуры складывать:

```text
страницы
вид
брошюра
навигация
экспорт
warning
```

Правильно разделять:

```text
Документ
Страницы
Вид
Брошюра
Навигация
Экспорт
```

---

### 9.7. Нельзя делать неработающие псевдо-кнопки

Если элемент выглядит как кнопка или меню — он должен работать.

Пример ошибки:

```text
⋮⋮ в колонке страниц
```

Он выглядел как меню, но был только индикатором drag-and-drop.  
Так нельзя.

Если меню нет — не показывать три точки.

---

## 10. Как правильно делать следующие изменения

Перед любым изменением:

1. Найти текущую реализацию.
2. Понять, какой слой меняется:
   - логика;
   - React JSX;
   - CSS;
   - DOM-layer `album-layers`;
   - экспорт;
   - booklet engine.
3. Удалить конфликтующие старые правила.
4. Не добавлять новую заплатку поверх старой.
5. Проверить, что нет дублей.

## 11. Мини-чеклист перед выдачей ZIP

Перед тем как отдавать ZIP, проверить:

```text
npm run build
```

Потом текстово проверить:

```text
нет src/patch20-top-panels.css
нет vite string-replace hotfix
нет UI cleanup pass 2/3
нет Toggle cleanup nth-child псевдо-подписей
нет кнопок с кавычками типа 'Направляющие'
нет ⋮⋮ в page rail
```

Проверить в интерфейсе:

```text
+ Страница есть
+ Пустая есть
Копия одна
Удалить одна
Направляющие не называются Сетка
Сетка окон отдельная кнопка
Текстовый режим не показывает фото-панель
Брошюра не превращает экспорт в огромные плитки
```


Редактор должен быть не кучей заплаток, а понятной системой.
