# Collage Creator

Веб-редактор фотоальбомов и печатных брошюр. Проект поддерживает многостраничные макеты, свободные и сеточные фото-окна, текст, линии, пользовательские шаблоны, локальные/облачные сохранения и PNG/ZIP-экспорт для печати.

## Требования

- Node.js 22
- npm
- PostgreSQL для аккаунтов и облачных проектов

```bash
nvm use
npm ci
npm run dev
```

Production-проверка:

```bash
npm run lint
npm test
npm run build
npm start
```

## Переменные окружения

| Переменная | Назначение | Значение по умолчанию |
|---|---|---|
| `DATABASE_URL` | Подключение PostgreSQL | нет |
| `SESSION_SECRET` | Подпись сессионных cookie | случайный секрет на каждый запуск в production |
| `JSON_LIMIT_BYTES` | Максимальный размер проекта в API | 60 МБ |
| `AUTH_JSON_LIMIT_BYTES` | Максимальный размер запроса входа/регистрации | 16 КБ |
| `MAX_PROJECTS_PER_USER` | Лимит проектов пользователя | 25 |
| `MAX_USER_STORAGE_BYTES` | Общий лимит облачного хранилища | 500 МБ |
| `AUTH_WINDOW_MS` | Окно подсчёта неудачных входов | 15 минут |
| `AUTH_MAX_ATTEMPTS` | Число неудачных попыток до блокировки | 20 |
| `AUTH_BLOCK_MS` | Время блокировки | 15 минут |
| `AUTH_MAX_TRACKED_KEYS` | Максимум записей временного rate limit | 10 000 |
| `TRUST_PROXY` | Разрешить использование `X-Forwarded-For` | `false` |

На Railway задайте постоянный `SESSION_SECRET`. `TRUST_PROXY=true` включайте только когда входящий заголовок формирует доверенный reverse proxy.

## Хранение проекта

Один снимок проекта используется для всех целей сохранения:

- `localStorage` — совместимость и аварийный fallback;
- IndexedDB — основное локальное сохранение больших проектов;
- PostgreSQL — облачный проект авторизованного пользователя;
- JSON — ручная резервная копия.

Открытие локального JSON или локального сохранения отвязывает макет от предыдущего облачного ID. Следующее облачное сохранение должно создать новый проект, а не перезаписать ранее открытый.

Перед рискованными изменениями редактора проверяйте сценарий:

1. собрать страницу в свободном режиме;
2. сохранить JSON;
3. повторно открыть;
4. убедиться, что координаты, размеры, кадрирование, текст и линии не изменились.

## Структура

- `src/AppLive.jsx` — React/Konva-интерфейс редактора;
- `src/editor/` — чистые модели страниц, рамок, фото, брошюр и загрузки проектов;
- `public/cloud-auth.js` — панель аккаунта и облачных проектов;
- `public/project-storage.js` — IndexedDB и миграция локальных сохранений;
- `server.js` — HTTP/API-сервер;
- `server/` — квоты, безопасность авторизации, чтение запросов и статические файлы.

## CI

Каждый push и pull request в `main` выполняет:

1. `npm ci`
2. `npm run lint`
3. `npm test`
4. `npm run build`

Не сливайте изменения редактора при красном тесте или без production build.

## Печать

PNG и ZIP являются растровыми файлами. Перед печатью проверяйте физический размер, масштаб 100%, порядок лицевых/оборотных сторон и отсутствие технических направляющих в чистовом макете. JSON проекта храните отдельно от печатных файлов.

## Лицензии ресурсов

Перед коммерческим распространением необходимо проверить и документировать лицензии всех файлов в `public/fonts` и `public/template-assets`. Добавляйте подтверждения лицензий в отдельный `THIRD_PARTY_LICENSES.md`.
